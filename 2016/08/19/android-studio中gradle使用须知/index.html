android studio中gradle使用须知

发表于 2016-08-19   |   分类于 android   |  

Gradle是个好东西啊，只是Android Studio都用了好长时间了，才慢慢搞清楚是这么回事。

Gradle是个牛逼的构建工具，更多的东西了解下就够了，反正是比Make，Ant好用，也快速的。这就是为什么不用eclipse，改用as的原因。gradle使用Groovy语言，但是这语言我可不懂，但是一些重要的语句，反正弄明白就行了。

Gradle在哪儿呢

Android Studio安装好后（mac下），gradle就集成在包里面了。路径大概长这样：
/Applications/Android Studio.app/Contents/gradle/gradle-2.14.1
然后在用户目录下还有生成了一个.gradle的目录，里面也是gradle的文件，这个实际是命令行下使用的gradle，在Android Studio中将这个目录称为 Service Directory Path。这也是为什么第一次使用./gradlew命令的时候，首先会检查这里面是否已经安装有相应版本的gradle，否则会重新进行下载。

gradle的版本

在工程目录下，gradle/wrapper/gradle-wrapper.properties文件中指定了使用的版本号。

distributionUrl=https\://services.gradle.org/distributions/gradle-2.14.1-all.zip

所以我们在导入别人的项目时候，可以先将这里的版本号改成自己本机的，否则就要等半天了去下载了。

Gradle Wrapper是搞毛的啊

Gradle Wrapper ，意为Gradle的包装。本意是用来确保项目使用自身设定gradle版本进行构建，而不是一定要使用本机安装的版本。我们强制在gradle-wrapper.properties文件中修改版本，是有可能出错的。

到底用的是哪个Gradle

首先看这个图，as中gradle设置。

  1. 使用本机版本，就是local gradle的版本。
  2. 使用gradle wrapper，这是推荐的方式。它用的是Service Path下的gradle。
  3. 如果通过命令行执行./gradlew 命令，使用的是gradle wrapper。

如果gradle的版本号不存在都会导致重新下载，这个过程很漫长。这下就清楚了。下图就是我的.gradle目录已经存在的版本。

这时候，如果别人的项目使用的是2.10或者2.14，我都可以直接使用了。

gradle的几个常用命令

./gradlew -v 版本号

./gradlew clean 清除9GAG/app目录下的build文件夹

./gradlew build 检查依赖并编译打包

这里注意的是 ./gradlew build 命令把 debug、release 环境的包都打出来，如果正式发布只需要打 Release 的包，该怎么办呢，下面介绍一个很有用的命令 assemble , 如

./gradlew assembleDebug 编译并打Debug包

./gradlew assembleRelease 编译并打Release的包

build.gradle

这里面可以自己加入一些语句，比如设置打包出来的apk名字。通过配置Flavor来生成不同渠道的apk。

  • 配置名字，放在release里面。

    applicationVariants.all { variant ->

        variant.outputs.each { output ->
        def outputFile = output.outputFile
        if (outputFile != null && outputFile.name.endsWith('.apk')) {
            def fileName = "yuqing_${variant.productFlavors[0].name}_v${defaultConfig.versionName}.apk"
            output.outputFile = new File(outputFile.parent, fileName)
        }
    }
}

  • 配置不同Flavor。这里面的门道就多了。简单的说，如果需要打出不同包名，版本的apk都可以在此设置。并且程序中可以通过BuildConfig直接获取里面的配置，从而做出不同的判断。

关于生成多渠道包，美团的blog中分享的非常详细。http://tech.meituan.com/mt-apk-adaptation.html

  • 加入so文件

    sourceSets {  
main {  
    jniLibs.srcDirs = ['libs']  
}  
}
