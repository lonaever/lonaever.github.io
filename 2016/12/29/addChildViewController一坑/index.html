addChildViewController一坑

发表于 2016-12-29   |   分类于 ios   |  

又是一个让人匪夷所思的问题，让人怀疑人生！

问题的产生

ios上，对于需要登录后才能进入主界面的应用，我通常是弄了一个LoginViewController，在MainViewController的ViewDidLoad中进行显示。可以当MainViewController是UINavigationController就出现问题了。

比如：

- (void)viewDidLoad {
    [super viewDidLoad];
    // Do any additional setup after loading the view.
    [self showLogin];
}

- (void)showLogin
{
    LoginViewController *login=[self.storyboard instantiateViewControllerWithIdentifier:@"Login"];
    [self addChildViewController:login];
    [login willMoveToParentViewController:self];
    login.view.frame=self.view.bounds;
    [self.view addSubview:login.view];
    [login didMoveToParentViewController:self];
}

可是，当MainViewController是TabController或者普通ViewController的时候，都没有问题。为UINavigationController的时候，这样的加进去就出问题了，显示出来的view，始终有个navbar在顶部。并且移除loginview的时候，移除不掉。

比如LoginViewController执行：

[self.view removeFromSuperview];
[self removeFromParentViewController];

屏幕显示一个导航栏，下面就全黑了。

暂时的解决办法

只能放在- (void)viewDidAppear:(BOOL)animated中去执行。
