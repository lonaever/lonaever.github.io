android中so文件无法加载问题

发表于 2017-02-07   |   分类于 android   |  

大坑就这么出现了

异常如：java.lang.UnsatisfiedLinkError: Native method not found: com.autonavi.amap.mapcore.MapCore.nativeNewInstance

这可是一直都好好运行着的高德地图啊。就因为增加了一个三方库，就变成这样了，地图死活无法加载。

原因

说到底就是so动态链接文件加载不到造成的。而so文件是按照cpu类型进行依次加载的

但是我们的项目中时常包含多种三方库，比如我的就包含了高德，极光，和pdf阅读。但是每个库提供的cpu版本并不是一致的，如果相差了就会在那样的机型上出现加载不到异常。

解决思路

删除多余的cpu类型版本，留下大家都有的，比如就留下armeabi和arm64-v8a。

但是仍然会有个麻烦，比如pdf阅读插件是通过gradle直接包含进来的，我没法删除）

  • 这时候，还可以通过改名字。把没有的类型通通复制一遍armeabi进去

  • ####最好的解决办法，通过build.gradle进行配置就能搞定，方法如下：

    defaultConfig {
...

ndk {
    //选择要添加的对应cpu类型的.so库。
    abiFilters 'armeabi', 'armeabi-v8a'
    // 还可以添加 'x86', 'x86_64', 'mips', 'mips64', 'armeabi-v7a'
}
}
