node下环境变量的使用

发表于 2018-09-07   |   分类于 node   |  

使用Express框架的时候，对于环境变量常看见是这样使用的：

  1. app.js里面有这样的写法

    res.locals.message = err.message;
  res.locals.error = req.app.get('env') === 'development' ? err : {};

  2. bin/www里面式样写的

    /**
 * Get port from environment and store in Express.
 */

var port = normalizePort(process.env.PORT || '3000');
app.set('port', port);

  3. express文档上有这么写

    启动这个应用（MacOS 或 Linux 平台）：
$ DEBUG=myapp npm start
Windows 平台使用如下命令：
> set DEBUG=myapp & npm start

process.env的用法

默认是通过环境变量中的NODE_ENV来控制的。配置NODE_ENV有好几个办法。

  1. 通过启动命令中去设置，只对当前应用有效。比如可以直接在package.json中去这样配置，scripts。

    "scripts": {
    "start": "PORT=3000 node ./bin/www",
    "production": "NODE_ENV=production PORT=3000 node ./bin/www",
    "doc": "apidoc -i ./routes/ -o ./apidoc/"
  },

然后通过npm run production就是生产环境的启动啦。

  1. 通过linux下的环境变量配置来直接设置NODE_ENV。
  • export NODE_ENV=PRODUCTION 只针对当前shell有效。
  • 编辑当前用户目录下的.bash_profile，export NODE_ENV=production
  • 对所有用户生效编辑/etc/profile，添加export NODE_ENV=production
  1. windows环境变量，右键电脑属性中。临时配置通过启动命令中，增加set PORT=8080

Linux挂载磁盘和分区备忘

发表于 2018-04-17   |   分类于 Linux   |  

很少操作服务器，这些命令不常用，备忘一下。

本次使用微软Azure创建的虚机，然后添加了500G的数据磁盘。添加完成后，需要登录到虚机上完成分区。

1. 查看磁盘

sudo fdisk -l

2. 进入分区

sudo fdisk /dev/sdc

Command (m for help): n
Partition type:
   p   primary (0 primary, 0 extended, 4 free)
   e   extended
Select (default p): p
Partition number (1-4, default 1): 1
First sector (2048-10485759, default 2048):
Using default value 2048
Last sector, +sectors or +size{K,M,G} (2048-10485759, default 10485759):
Using default value 10485759

Last sector用于指定分区的大小，比如输入+50G，就表示分50G的大小。

重复此步骤可以创建出多个分区。

3. 格式化

sudo mkfs -t ext4 /dev/sdc1

4. 挂载

sudo mkdir /datadrive
sudo mount /dev/sdc1 /datadrive

5. 启动引导

推荐使用uuid的方式

sudo -i blkid 查看uuid

编辑配置文件

sudo vi /etc/fstab
UUID=33333333-3b3b-3c3c-3d3d-3e3e3e3e3e3e   /datadrive   ext4   defaults,nofail   1   2

后面1和2，分别代表1是dump可以识别为需要备份的区域。2是个启动检查什么的玩意。

CSS3选择器笔记

发表于 2018-03-14   |   分类于 CSS   |  

属性选择器

基本语法

[xxx=xxx]{}

CSS3中新增3个具有模糊匹配的写法

  • [xxx*=xxx]{} 相当于contain
  • [xxx^=xxx]{} 相当于startWith
  • [xxx$=xxx]{} 相当于endWith

伪元素选择器

诸如：

  • p:first-line{}
  • p:first-letter{}
  • p:before{} 在p前面加入一些字符
  • p:after{ content: "--"}

伪类选择器

  • :root{} 表示整个根页面。如果设置了这个属性，再设置body就只表示整个内容部分了。
  • :empty{} 可以选择到所有空内容的标签，比如<td></td>
  • div *:not(h1){} 表示div下的不是h1的标签

CentOS下搭建yum本地环境

发表于 2017-09-14   |  

环境

当系统的环境中没有外网的时候，这个事情就很有必要了。不然全部通过rpm来安装会累死人的。在此记录一下。

通过ISO镜像来制作源

  1. 首先复制一个镜像文件至服务器上。存放在/mnt/iso/Centos.iso
  2. 创建一个挂载目录 mkdir /mnt/centos
  3. 添加到自动运行中
    vi /etc/fstab
    追加下面一行
    /mnt/CentOS-7.0-1406-x86_64-DVD.iso /mnt/centos iso9660 defaults,ro,loop 0 0
  4. 执行全量命令，重新挂载mount -a

  5. vi /etc/yum.repos.d/CentOS-Local.repo 新建本地源配置文件
    添加下面几行

    [local]
name=local
baseurl=file:///mnt/centos
enabled=1
gpgcheck=1
gpgkey=file:///mnt/centos/RPM-GPG-KEY-CentOS-7

接着重建下yum源缓存
yum clean yum repolist

6、可以把/etc/yum.repo.d/下的其他文件都删了。
