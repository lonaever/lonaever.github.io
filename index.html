使用cocoapods

发表于 2016-08-24   |   分类于 ios   |  

重装系统后，安装cocoapods竟然冒出那么多问题，真是奇了怪了。主要是环境问题。

升级ruby

问题描述：

执行sudo gem install cocoapods 后提示

ERROR: Error installing cocoapods:
activesupport requires Ruby version >= 2.2.2.

解决办法

  1. 安装rvm（ruby的管理器），用它来安装新版本的ruby。

    curl -L get.rvm.io | bash -s stable

    安装完成后提示：

    * WARNING: You have '~/.profile' file, you might want to load it,
to do that add the following line to '/Users/minyu/.bash_profile':

  source ~/.profile

    就是需要在.bash_profile文件中增加一行。

  2. 这时候可以查看ruby的版本，和可以安装的版本。

    rvm -v
rvm 1.27.0 (latest) by Wayne E. Seguin <wayneeseguin@gmail.com>, Michal Papis     <mpapis@gmail.com> [https://rvm.io/]
rvm list known

  3. 安装2.3

    rvm install 2.3

    这时候，会自动先安装homebrew，真是麻烦。

    About to install Homebrew, press `Enter` for default installation in `/usr/local`,
type new path if you wish custom Homebrew installation (the path needs to be writable for user)

    （安装过程真的很慢！还要complie ruby2.3）

升级gem

更改ruby的源

为了不翻墙，为了速度。照做吧！

gem sources --remove https://rubygems.org/
gem sources -a https://ruby.taobao.org/
gem sources -l

然后再执行

sudo gem update --system
gem -v

安装cocoapods

现在一切就绪，安装

sudo gem install cocoapods
初始化索引

这个步骤实在太慢，据说大概有400M的东西需要下载。而我的情况就是无休止的出现：

pod setup

[!] /usr/bin/git clone https://github.com/CocoaPods/Specs.git master

Cloning into 'master'...
error: RPC failed; curl 56 SSLRead() return error -9806
fatal: The remote end hung up unexpectedly
fatal: early EOF
fatal: index-pack failed

通过终端可以查看进度：

cd ~/.cocoapods/
du -sh *
替换pod源
pod repo remove master
pod repo add master 镜像源地址
pod repo update
pod setup

简单使用

  1. 创建Podfile

    pod init

    使用命令创建，然后再修改比较好。现在的版本是要求一定要有target的。如：

    # Uncomment this line to define a global platform for your project
platform :ios, '8.0'

target 'TestHealthKit' do
  # Uncomment this line if you're using Swift or would like to use dynamic frameworks
  # use_frameworks!

  # Pods for TestHealthKit
  pod 'YYKit', '~> 1.0.7'

  target 'TestHealthKitTests' do
inherit! :search_paths
# Pods for testing
  end

  target 'TestHealthKitUITests' do
inherit! :search_paths
# Pods for testing
  end

end

  2. 执行pod install,pod update，进行第一次安装依赖和升级。带上参数 --no-repo-update可以避免去更新索引库。

android拍照问题（i9300）

发表于 2016-08-22   |   分类于 android   |  

android上的拍照及自定义相册，现在直接使用三方库处理就好，没那么多问题。如：https://github.com/pengjianbo/GalleryFinal 功能很强大。大多数应用场景这个库都能满足了，自己只需要去实现图片压缩，然后上传就行。

过去就麻烦了，问题很多，特别是在三星i9300上简直就是扯淡，至今记忆犹新。

常见问题

  • i9300上activity销毁问题

    i9300上调用相机后，调用的Activity会立即被destory

    解决办法：

    1. 首先xml上增加一句

      android:configChanges="keyboardHidden|orientation|screensize"

      否则拍照结果无法获取。

    2. 通过重写Activity的 onSaveInstanceState() 和 onRestoreInstanceState()方法。将需要记录的数据手动保存下来，在重新创建的时候，进行恢复。
